Explain Python's GIL (Global Interpreter Lock). How does it affect multi-threaded applications, and how did you handle concurrency in your ETL pipelines?
ConcurrencyAdvanced
▲ Hide
Model Answer
The GIL is a mutex in CPython that allows only one thread to execute Python bytecode at a time, even on multi-core systems. This means CPU-bound tasks don't benefit from threading. For I/O-bound tasks (like network calls, file reads), threading still helps because the GIL is released during I/O waits.

In my ETL pipelines at Citi:

  • Used asyncio + aiohttp for concurrent API calls during data ingestion
  • Apache Airflow's DAG parallelism handled task-level concurrency across pipeline stages
  • For PySpark at MSCI, distributed processing across worker nodes eliminated GIL concerns entirely

What are Python decorators? Write a decorator that logs execution time and handles exceptions for any function.
DecoratorsCoding
▼ Show
Model Answer
A decorator is a higher-order function that wraps another function to extend its behavior without modifying it. They use the @ syntax and are heavily used in Django/Flask for authentication, caching, and logging.

Example:
import time, logging, functools
def timed_logger(func):
  @functools.wraps(func)
  def wrapper(*args, **kwargs):
    start = time.perf_counter()
    try:
      result = func(*args, **kwargs)
      logging.info(f"{func.__name__} took {time.perf_counter()-start:.3f}s")
      return result
    except Exception as e:
      logging.error(f"{func.__name__} failed: {e}")
      raise
  return wrapper

Explain the transformer architecture. What are attention mechanisms and why are they important for LLMs?
Deep LearningTheory
▼ Show
Model Answer
Transformers use self-attention to weigh the importance of each token relative to all others in a sequence — enabling parallel processing (unlike RNNs which are sequential).

Self-attention: For each token, compute Query (Q), Key (K), Value (V) vectors. Attention score = softmax(QK^T / √d_k) × V. This lets the model focus on relevant context regardless of distance in the sequence.

Multi-head attention: Multiple attention heads learn different relationship types simultaneously (syntactic, semantic, coreference).

Why important for LLMs: Enables understanding of long-range dependencies — critical for regulatory documents where a clause on page 50 may reference a definition on page 5. My 768-dim embeddings are the output of transformer encoder layers.

How do you implement blue-green or canary deployments? Why are they important for financial applications?

▼ Show
Model Answer
Blue-Green: Two identical production environments. Deploy to "green" while "blue" serves traffic. Switch traffic instantly. Instant rollback by switching back.

Canary: Gradually route a percentage of traffic to the new version (e.g., 5% → 25% → 100%). Monitor metrics at each stage. Roll back if issues detected.

Why critical in finance:
  • Zero-downtime deployments — financial systems must be always available
  • Risk mitigation — a bug in model calculations affects only 5% of users initially
  • Regulatory requirement — changes must be validated before full rollout
  • Instant rollback — if a new model version produces incorrect outputs, revert in seconds
In K8s: canary deployments implemented with Argo Rollouts or Istio traffic splitting.
